# oss-signal v0.8.0

`oss-signal` v0.8.0 adds a no-fail workflow generator and a maintainer feedback path for third-party trial responses.

## Added

- `--format workflow` renders a copyable GitHub Actions workflow that runs `SalmonPlays/oss-signal@v0.8.0` in report-only mode.
- Trial feedback issue template for maintainers who reviewed or tried a report but did not adopt the tool.
- Maintainer feedback documentation that explains how neutral, negative, or out-of-scope responses are counted.

## Verification

```bash
npm run check
node src/cli.js . --format workflow --output docs/examples/maintainer-trial-workflow.yml
```

The workflow output is intentionally non-gating. It writes a step summary and uploads `oss-signal-report.md` without setting `fail-under`.
