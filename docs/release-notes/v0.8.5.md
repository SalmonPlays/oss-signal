`oss-signal` v0.8.5 improves the first-run maintainer experience.

## Changes

- Markdown reports now include an `Evidence / next step` column.
- Passed checks show the detected file or workflow evidence.
- Failed checks show the concrete missing maintainer-readiness action in the checks table.
- README now starts with a 30-second quickstart before the reviewer evidence index.
- Added [docs/quickstart.md](../quickstart.md) for CLI, issue, plan, and no-fail workflow trials.

## Verification

- `npm run check`
- `npm publish --dry-run`
